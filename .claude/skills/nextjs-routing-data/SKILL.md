# Next.js App Router - Routing & Data Fetching con TypeScript

Guía completa para implementar routing, data fetching y arquitectura server/client en Next.js 13+ con App Router, TypeScript y mejores prácticas de producción.

## ¿Qué es Next.js App Router?

Next.js 13+ introdujo **App Router** (`/app` directory) con features revolucionarias:
- ✅ **Server Components**: React Server Components por defecto (fetch en servidor)
- ✅ **File-based Routing**: Carpetas = rutas, convenciones especiales (page.tsx, layout.tsx)
- ✅ **Layouts Anidados**: UI persistente con carga parcial
- ✅ **Streaming**: Suspense boundaries para progressive rendering
- ✅ **Data Fetching**: async/await directo en componentes (no getServerSideProps)
- ✅ **Route Handlers**: API endpoints con tipado fuerte (route.ts)

**Diferencia con Pages Router:** App Router usa Server Components por defecto, fetch nativo con cache, y arquitectura server-first.

```bash
# Crear proyecto Next.js 14+ con App Router
npx create-next-app@latest my-app --typescript --app --tailwind
```

## File-Based Routing

### Convenciones de Carpetas

```
app/
├── page.tsx                    # / (Home)
├── about/
│   └── page.tsx                # /about
├── blog/
│   ├── page.tsx                # /blog
│   └── [slug]/
│       └── page.tsx            # /blog/my-post
├── dashboard/
│   ├── layout.tsx              # Layout compartido
│   ├── page.tsx                # /dashboard
│   ├── settings/
│   │   └── page.tsx            # /dashboard/settings
│   └── users/
│       └── [id]/
│           └── page.tsx        # /dashboard/users/123
└── api/
    └── users/
        └── route.ts            # /api/users
```

### page.tsx - Rutas Públicas

```tsx
// app/page.tsx (Home)
export default function HomePage() {
  return (
    <div>
      <h1>Welcome to Next.js</h1>
    </div>
  );
}

// app/about/page.tsx
export default function AboutPage() {
  return (
    <div>
      <h1>About Us</h1>
    </div>
  );
}
```

### Dynamic Routes - [param]

```tsx
// app/blog/[slug]/page.tsx
interface BlogPostPageProps {
  params: { slug: string };
}

export default async function BlogPostPage({ params }: BlogPostPageProps) {
  const { slug } = params;

  // Fetch data directo (Server Component)
  const post = await fetch(`https://api.example.com/posts/${slug}`).then((r) => r.json());

  return (
    <article>
      <h1>{post.title}</h1>
      <p>{post.content}</p>
    </article>
  );
}
```

### Catch-All Routes - [...param]

```tsx
// app/docs/[...slug]/page.tsx
interface DocsPageProps {
  params: { slug: string[] };
}

export default function DocsPage({ params }: DocsPageProps) {
  // /docs/intro → slug: ['intro']
  // /docs/api/authentication → slug: ['api', 'authentication']
  const path = params.slug.join('/');

  return <div>Documentation: {path}</div>;
}
```

### Optional Catch-All - [[...param]]

```tsx
// app/shop/[[...category]]/page.tsx
interface ShopPageProps {
  params: { category?: string[] };
}

export default function ShopPage({ params }: ShopPageProps) {
  // /shop → category: undefined
  // /shop/electronics → category: ['electronics']
  // /shop/electronics/phones → category: ['electronics', 'phones']

  const categoryPath = params.category?.join(' > ') || 'All Products';

  return <div>Category: {categoryPath}</div>;
}
```

## Server Components vs Client Components

### Server Components (Default)

```tsx
// app/blog/page.tsx (Server Component por defecto)
async function getPosts() {
  const res = await fetch('https://api.example.com/posts', { cache: 'no-store' });
  return res.json();
}

export default async function BlogPage() {
  const posts = await getPosts();

  return (
    <div>
      <h1>Blog Posts</h1>
      {posts.map((post) => (
        <article key={post.id}>
          <h2>{post.title}</h2>
        </article>
      ))}
    </div>
  );
}
```

**Beneficios:**
- ✅ Fetch directo (async/await)
- ✅ Acceso a backend (DB, filesystem, secrets)
- ✅ Bundle size más pequeño (código en servidor)
- ✅ SEO optimizado (HTML pre-renderizado)

### Client Components ('use client')

```tsx
// app/components/Counter.tsx
'use client';

import { useState } from 'react';

export default function Counter() {
  const [count, setCount] = useState(0);

  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={() => setCount(count + 1)}>Increment</button>
    </div>
  );
}
```

**Cuándo usar 'use client':**
- ✅ Interactividad (onClick, onChange)
- ✅ React hooks (useState, useEffect, useContext)
- ✅ Browser APIs (localStorage, window, document)
- ✅ Event listeners
- ✅ Custom hooks

### Composición Server + Client

```tsx
// app/page.tsx (Server Component)
import Counter from './components/Counter'; // Client Component

async function getData() {
  const res = await fetch('https://api.example.com/data');
  return res.json();
}

export default async function HomePage() {
  const data = await getData(); // Server-side fetch

  return (
    <div>
      <h1>Server Data: {data.title}</h1>

      {/* Client Component embebido */}
      <Counter />
    </div>
  );
}
```

**Patrón:** Server Component como wrapper, Client Component solo donde se necesita interactividad.

## Data Fetching Patterns

### Fetch con Cache

```tsx
// app/posts/page.tsx
async function getPosts() {
  // Cache indefinidamente (regenera en build)
  const res = await fetch('https://api.example.com/posts', { cache: 'force-cache' });
  return res.json();
}

export default async function PostsPage() {
  const posts = await getPosts();
  return <div>{/* render posts */}</div>;
}
```

### Fetch con Revalidación

```tsx
// Revalidar cada 60 segundos
async function getPosts() {
  const res = await fetch('https://api.example.com/posts', {
    next: { revalidate: 60 },
  });
  return res.json();
}
```

### Fetch sin Cache

```tsx
// Siempre fresh data (equivalente a getServerSideProps)
async function getUser() {
  const res = await fetch('https://api.example.com/user', { cache: 'no-store' });
  return res.json();
}
```

### Parallel Requests

```tsx
async function getPageData() {
  // ✅ Paralelo: más rápido
  const [user, posts, comments] = await Promise.all([
    fetch('/api/user').then((r) => r.json()),
    fetch('/api/posts').then((r) => r.json()),
    fetch('/api/comments').then((r) => r.json()),
  ]);

  return { user, posts, comments };
}
```

### Sequential Requests (Waterfall)

```tsx
async function getUserPosts(userId: string) {
  // ❌ Sequential: más lento (waterfall)
  const user = await fetch(`/api/users/${userId}`).then((r) => r.json());
  const posts = await fetch(`/api/users/${user.id}/posts`).then((r) => r.json());

  return { user, posts };
}
```

**Tip:** Usa Promise.all cuando los requests son independientes.

### Streaming con Suspense

```tsx
// app/dashboard/page.tsx
import { Suspense } from 'react';

async function Analytics() {
  const data = await fetch('/api/analytics', { cache: 'no-store' }).then((r) => r.json());
  return <div>Analytics: {data.value}</div>;
}

async function Revenue() {
  const data = await fetch('/api/revenue', { cache: 'no-store' }).then((r) => r.json());
  return <div>Revenue: ${data.total}</div>;
}

export default function DashboardPage() {
  return (
    <div>
      <h1>Dashboard</h1>

      {/* Cada Suspense boundary carga independientemente */}
      <Suspense fallback={<div>Loading analytics...</div>}>
        <Analytics />
      </Suspense>

      <Suspense fallback={<div>Loading revenue...</div>}>
        <Revenue />
      </Suspense>
    </div>
  );
}
```

**Beneficio:** Streaming progresivo (no bloquea toda la página)

## Route Handlers - API Endpoints

### GET Request

```tsx
// app/api/posts/route.ts
import { NextResponse } from 'next/server';

export async function GET() {
  const posts = await fetch('https://api.example.com/posts').then((r) => r.json());

  return NextResponse.json(posts);
}
```

### POST Request con Body

```tsx
// app/api/posts/route.ts
import { NextRequest, NextResponse } from 'next/server';

interface CreatePostBody {
  title: string;
  content: string;
}

export async function POST(request: NextRequest) {
  const body: CreatePostBody = await request.json();

  const newPost = await fetch('https://api.example.com/posts', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify(body),
  }).then((r) => r.json());

  return NextResponse.json(newPost, { status: 201 });
}
```

### Dynamic Route Handler

```tsx
// app/api/posts/[id]/route.ts
import { NextRequest, NextResponse } from 'next/server';

interface RouteParams {
  params: { id: string };
}

export async function GET(request: NextRequest, { params }: RouteParams) {
  const { id } = params;

  const post = await fetch(`https://api.example.com/posts/${id}`).then((r) => r.json());

  return NextResponse.json(post);
}

export async function DELETE(request: NextRequest, { params }: RouteParams) {
  const { id } = params;

  await fetch(`https://api.example.com/posts/${id}`, { method: 'DELETE' });

  return NextResponse.json({ success: true });
}
```

### Error Handling en Route Handlers

```tsx
// app/api/users/[id]/route.ts
import { NextRequest, NextResponse } from 'next/server';

export async function GET(request: NextRequest, { params }: { params: { id: string } }) {
  try {
    const user = await fetch(`https://api.example.com/users/${params.id}`).then((r) => {
      if (!r.ok) throw new Error('User not found');
      return r.json();
    });

    return NextResponse.json(user);
  } catch (error) {
    return NextResponse.json({ error: 'User not found' }, { status: 404 });
  }
}
```

### useRouter - Navegación Programática

```tsx
// app/components/LoginForm.tsx
'use client';

import { useRouter } from 'next/navigation';

export default function LoginForm() {
  const router = useRouter();

  const handleLogin = async () => {
    // Navegar después de login
    router.push('/dashboard');

    // Navegar sin agregar a history
    router.replace('/dashboard');

    // Navegar hacia atrás
    router.back();

    // Refrescar ruta actual
    router.refresh();
  };

  return <button onClick={handleLogin}>Login</button>;
}
```

### usePathname - Ruta Actual

```tsx

```

### redirect - Server-Side Redirect

### Dynamic Metadata

```tsx
// app/blog/[slug]/page.tsx
import type { Metadata } from 'next';

interface BlogPostPageProps {
  params: { slug: string };
}

export async function generateMetadata({ params }: BlogPostPageProps): Promise<Metadata> {
  const post = await fetch(`https://api.example.com/posts/${params.slug}`).then((r) => r.json());

  return {
    title: post.title,
    description: post.excerpt,
    openGraph: {
      title: post.title,
      description: post.excerpt,
      images: [post.image],
    },
  };
}

export default async function BlogPostPage({ params }: BlogPostPageProps) {
  const post = await fetch(`https://api.example.com/posts/${params.slug}`).then((r) => r.json());
  return <article>{post.content}</article>;
}
```

## generateStaticParams - Static Generation

```tsx
// app/blog/[slug]/page.tsx
export async function generateStaticParams() {
  const posts = await fetch('https://api.example.com/posts').then((r) => r.json());

  return posts.map((post) => ({
    slug: post.slug,
  }));
}

export default async function BlogPostPage({ params }: { params: { slug: string } }) {
  const post = await fetch(`https://api.example.com/posts/${params.slug}`).then((r) => r.json());
  return <article>{post.title}</article>;
}
```

**Beneficio:** Pre-renderiza todas las páginas en build time.

## Mejores Prácticas

### ✅ Arquitectura
1. **Server Components first** - Usar 'use client' solo cuando necesario
2. **Colocación de data fetching** - Fetch en el componente más cercano que necesita los datos
3. **Parallel fetching** - Promise.all para requests independientes
4. **Streaming con Suspense** - Para UX optimizada (no bloquear página completa)
5. **Layout anidados** - Para compartir UI y reducir re-renders

### ✅ Performance
1. **Static Generation** - generateStaticParams para rutas conocidas
2. **Cache estratégico** - Usar { next: { revalidate } } para ISR
3. **Prefetching** - Link prefetch para rutas críticas
4. **Code splitting** - Lazy load Client Components con dynamic()
5. **Optimización de imágenes** - Usar next/image

### ✅ TypeScript
1. **Tipar params** - Interfaces para params de rutas dinámicas
2. **Tipar metadata** - Usar type Metadata de 'next'
3. **Tipar API responses** - Interfaces para bodies y responses
4. **Tipar Route Handlers** - NextRequest, NextResponse tipados

### ✅ SEO
1. **Metadata por ruta** - generateMetadata para dynamic metadata
2. **Open Graph tags** - Para social sharing
3. **Sitemap.xml** - Generar automáticamente
4. **robots.txt** - Configurar crawling
