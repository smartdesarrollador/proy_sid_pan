---
description: Create a Product Requirement Document for a new feature
---

Create a PRD for the requested feature using this structure:

1. **Overview** - What and why
2. **Goals & Success Metrics** - Measurable outcomes
3. **User Stories** - As a [role], I want [action], so that [benefit]
4. **Requirements**
   - Functional requirements (numbered)
   - Non-functional requirements
5. **Technical Approach** - High-level implementation plan
6. **Phases** - Sequential implementation phases with dependencies
7. **Out of Scope** - Explicit exclusions
8. **Open Questions** - Unresolved decisions

Save to `prd/` with a descriptive filename.

Use $ARGUMENTS as the feature description.
