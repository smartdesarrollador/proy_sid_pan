---
description: Review a pull request for quality, security, and correctness
---

Review the current branch's changes against the base branch:

1. Run `git diff main...HEAD` to see all changes
2. Analyze each changed file for:
   - Correctness and logic errors
   - Security vulnerabilities
   - Performance implications
   - Test coverage
3. Check that commit messages follow conventions
4. Provide a structured review with actionable feedback

Use severity levels: Critical, Warning, Suggestion, Praise.
